# c0rtex

a local, privacy-first personal ai assistant powered by [ollama](https://ollama.com). runs entirely on your hardware — no cloud, no telemetry, no subscriptions.

c0rtex is a modular system of python scripts that wraps ollama with tool calling, persistent memory, chat bridges, autonomous thinking sessions, and a web dashboard. it's designed for people who want a personal ai that actually knows them and runs on their own metal.

## features

- **interactive repl** with persistent conversation history and streaming responses
- **tool calling** — 30+ guardrailed tools for file ops, system monitoring, ctf/forensics, study tools, and homelab management
- **matrix bridge** — async chat bridge with image support, url auto-fetching, and digest queue
- **autonomous pondering** — scheduled thinking sessions that generate ideas, check system health, and maintain a knowledge base
- **news digest** — fetches rss feeds, ranks by relevance, summarizes top articles, queues the rest for on-demand delivery
- **morning briefings** — reads your schedule, reasons about dates with extended thinking, sends a daily update
- **deadline alerts** — parses dates from your school/project files, alerts when things are due soon
- **signal bridge** — responds to messages as you using a custom persona
- **web dashboard** — browse conversations, ponderings, digests, logs, and oura health data
- **single-shot mode** — pipe prompts in from cron jobs or scripts
- **structured logging** — every event logged as ndjson, viewable in the dashboard

## requirements

- **python 3.10+** (macOS users: `brew install python@3.11`)
- **ollama** — local llm inference
  - macOS: `brew install ollama`
  - linux: `curl -fsSL https://ollama.com/install.sh | sh`
- **(optional) [pinchtab](https://pinchtab.com)** — web browsing capabilities

## installation

```bash
git clone https://github.com/t3t5u0403/c0rtex.git
cd c0rtex
pip3 install -r requirements.txt
python3 scripts/setup.py
```

## quickstart

the setup wizard will:
1. ask your name and preferences to generate a personalized `SOUL.md`
2. configure your ollama endpoint
3. detect your GPU and recommend + pull the right model
4. create a custom `c0rtex` ollama model
5. optionally install [pinchtab](https://pinchtab.com) for web browsing
6. optionally set up matrix bridge and homelab integrations
7. install scripts and templates to `~/.c0rtex/`
8. generate a complete `.env` with all integration options

then start chatting:

```bash
python ~/.c0rtex/scripts/c0rtex.py
```

## usage

```bash
# interactive repl
python ~/.c0rtex/scripts/c0rtex.py

# single-shot (for cron/scripts)
python ~/.c0rtex/scripts/c0rtex.py "what's on my schedule today?"

# matrix bridge (long-running service)
python ~/.c0rtex/scripts/c0rtex_matrix.py

# autonomous pondering session
python ~/.c0rtex/scripts/c0rtex_ponder.py

# cybersecurity news digest
python ~/.c0rtex/scripts/c0rtex_digest.py

# morning briefing (sends to matrix)
python ~/.c0rtex/scripts/c0rtex_briefing.py

# deadline check (sends to matrix)
python ~/.c0rtex/scripts/c0rtex_deadlines.py

# web dashboard → http://127.0.0.1:5000
python ~/.c0rtex/scripts/c0rtex_web.py
```

### cron examples

```cron
# morning briefing at 8am
0 8 * * * python ~/.c0rtex/scripts/c0rtex_cron.py briefing

# deadline check at 8pm
0 20 * * * python ~/.c0rtex/scripts/c0rtex_cron.py deadlines

# news digest at 8am and 6pm
0 8,18 * * * python ~/.c0rtex/scripts/c0rtex_digest.py

# autonomous pondering 3x daily
0 9,14,21 * * * python ~/.c0rtex/scripts/c0rtex_ponder.py
```

## optional features

### web browsing

c0rtex can browse the web using [pinchtab](https://github.com/pinchtab/pinchtab). the setup wizard will offer to install it automatically via npm or homebrew.

**manual installation (if needed):**

```bash
# via npm (recommended — cross-platform)
npm install -g pinchtab

# via homebrew (macOS)
brew install pinchtab

# via install script (macOS/linux)
curl -fsSL https://pinchtab.com/install.sh | bash
```

**usage:**

```bash
# start pinchtab (in a separate terminal or background)
pinchtab  # runs on http://localhost:9867

# c0rtex will automatically use it for web browsing
```

if pinchtab isn't running, web browsing tools will fail gracefully with installation instructions.


### how it works

every script that talks to ollama follows the same pattern: send messages → ollama returns tool calls → execute tools → feed results back → repeat until final text response. tools are guardrailed — file operations are scoped to allowed directories, all writes create `.bak` backups, and there's no raw shell access.

all paths and configuration are centralized in `c0rtex_paths.py`. every script imports from it. secrets are loaded from `~/.c0rtex/.env` at import time.

## tools

| category | tools |
|----------|-------|
| file ops | `read_files`, `safe_write`, `safe_append`, `search_files`, `file_diff`, `file_history` |
| study | `compile_notes`, `deadline_check`, `summarize_document`, `generate_quiz` |
| system | `gpu_status`, `disk_status`, `service_check`, `network_check`, `process_list` |
| ctf/forensics | `analyze_binary`, `hex_peek`, `entropy_check`, `extract_strings`, `binwalk_scan` |
| self-management | `inbox_add`, `pondering_summary`, `idea_search`, `project_status`, `soul_read` |
| homelab | `truenas_status`, `tailscale_status`, `unifi_clients` |
| web | `browse_page`, `browse_search` |
| health | `oura_summary` |

all tools are defined in `c0rtex_tools.py` with strict scoping and no shell injection vectors.

## integrations

all integrations are optional. configure them in `~/.c0rtex/.env`:

| integration | what it does |
|-------------|-------------|
| **ollama** | local llm inference (required) |
| **matrix** | chat bridge with image support and digest queue |
| **signal** | sms bridge via signal-cli daemon |
| **truenas** | storage pool and dataset monitoring |
| **unifi** | network client listing |
| **oura ring** | sleep and readiness tracking (oauth2) |
| **pinchtab** | browser content extraction |

## dashboard

the web dashboard at `http://127.0.0.1:5000` provides:

- **overview** — today's stats, recent events, latest pondering/digest
- **conversations** — browse cli and matrix chat history
- **ponderings** — archive of autonomous thinking sessions
- **digests** — archive of news digest issues
- **logs** — filterable activity log viewer
- **signal** — signal bridge status and message history
- **oura** — health data and oauth2 connection management

## configuration

after running `setup.py`, edit `~/.c0rtex/.env` to add more integrations. see `.env.example` for all available options.

the personality is defined in `~/.c0rtex/data/SOUL.md` — edit it to change how c0rtex talks to you. the default voice is lowercase, dry, and direct.

models are configured per-script at the top of each file. the defaults are:
- `c0rtex` (custom modelfile) — interactive repl and cron
- `qwen3.5:27b`+ — matrix bridge, pondering, digest (setup wizard picks based on your GPU)
- same model for briefings and deadline analysis (extended thinking)

swap these for whatever fits your hardware.
